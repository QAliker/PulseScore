# PulseScore Arena — Design Context

## Users

- **Casual fan**: desktop glance, wants scores + status at a glance, low cognitive load.
- **Stats nerd**: same surface, but hungry for minute, xG, lineups, H2H. Scans dense data fast.
- **Device**: desktop-first MVP. Responsive required but no mobile-only flourishes yet.
- **Context**: match-day sessions, often 90+ min tab open. Screen may sit idle then spike on goal.
- **Scale**: 2 leagues at launch (football only). ~20–40 concurrent fixtures on peak matchday.

## Brand Personality

- **Three words**: clear, clean, understandable.
- **Positive refs**: FlashScore (density + speed), SofaScore (data depth + polish). Match their clarity, add our own sauce.
- **Anti-references**: ESPN (corporate-bloated, ad-heavy, loud). Avoid crypto-dashboard neon and generic admin-template card grids.
- **Emotional goal**: trust + speed. User believes score updated the instant it changed.

## Aesthetic Direction

- **Theme**: light + dark, both first-class. System preference default, user toggle.
- **Accent palette**: purple (primary brand) + green (live / pitch / positive state). Green doubles as pitch-grass texture motif.
- **Visual motif**: grass-pitch imagery tasteful — subtle texture on hero / league headers, NOT every card. Football-native without going costume.
- **Stack**: Next.js 16 App Router, React 19, Tailwind v4, shadcn/ui. OKLCH tokens. Container queries for feed cards.
- **Typography**: pair distinctive display with refined body. Avoid Inter / DM Sans / Syne. Look toward humanist sans + confident editorial sans or slab for numerals. Tabular figures mandatory for scores/minute.
- **Score animation**: flash + highlight on change. Moderate intensity — visible, not stadium-goal fireworks.

## Design Principles

1. **Score first, chrome last.** Score, minute, status readable in 200ms of scan. Decoration never obscures data.
2. **Density with breathing room.** FlashScore-level information, SofaScore-level polish. Tight groupings, generous league separators.
3. **Live state is loud, resolved state is quiet.** Live matches pull rank (top of feed, pulse, colored minute). Finished matches recede in weight.
4. **Trustworthy motion.** Flash on goal = confirmation, not celebration. Reduced-motion mode mutes to a color-only tick.
5. **No admin-card monoculture.** Avoid identical padded card grids. Feed is a rhythmic list, not a template.

# Feature 4 — Design Brief: Live Scores Frontend

## 1. Feature Summary

Real-time live-scores surface organized by league. Desktop-first feed where casual fans confirm a result in one glance and stats nerds drill into minute-by-minute state. Updates stream via WebSocket over an SSR-hydrated initial render.

## 2. Primary User Action

**See the current score of matches they care about, and trust it is current.** Everything else (filters, drill-down, stats) serves that core.

## 3. Design Direction

Editorial sports-paper meets trading terminal. Clear typographic hierarchy, tabular numerals, subtle pitch-grass texture reserved for league headers / hero band only. Purple = brand chrome (logo, active nav, focus rings). Green = live pulse, positive deltas, pitch motif. Neutrals tinted toward purple in both themes for cohesion.

Not neon. Not glass. Not corporate-ESPN. Not crypto-dashboard.

## 4. Layout Strategy

- **Shell**: top header (logo, search, theme toggle, user) + left sidebar (sport → league tree, football-only MVP but tree scales).
- **Main**: `LeagueFeed` — grouped sections by league. League header = sticky-in-section band with crest + name + live count + subtle pitch texture.
- **Feed rhythm**: live matches pinned at top of each league group (bright, pulsing minute). Scheduled next (muted). Finished last (quietest weight).
- **MatchCard**: horizontal row, NOT boxed card. Reserve boxed treatment for hero / featured match only. Row layout: `[time/minute] [home team + logo] [score] [away team + logo] [status pill] [expand chevron]`. Tabular score centered, weight 700, ~2× body size.
- **Asymmetry**: league header left-aligned, live badge right-aligned. No centering everything.
- **Responsive**: sidebar collapses to icon rail below 1024px, hamburger below 768px. Rows stay rows — do not collapse to cards.

## 5. Key States

| State | What user sees |
|---|---|
| **SSR initial** | Hydrated feed with last-known scores + timestamp "as of HH:MM". No layout shift on WS connect. |
| **Live connected** | Small green dot + "Live" label in header. Silent. |
| **Score change** | Score digit flashes green (home) / purple (away), row background highlights 600ms, then fades. Minute keeps pulsing. |
| **Match kicks off** | Row promotes from "scheduled" band to "live" band with slide transition (grid-template-rows, not height). |
| **Match ends** | Row demotes to "finished" band, minute replaced with "FT", weight drops. |
| **Loading** | Skeleton rows matching final layout. No spinner. |
| **Empty league** | "No matches today — next fixture Sat 15:00" with small pitch-line illustration. Teaches, not blanks. |
| **Error (HTTP)** | Inline banner at top of feed, retry button. Feed keeps last-known data. |
| **WS disconnected** | Header dot turns amber + "Reconnecting…" + countdown. Scores dim 10%, minute freezes with "⏸" glyph. Auto-reconnect with exp backoff. |
| **WS reconnected** | Dot flashes green, dim lifts, diff replays. |

## 6. Interaction Model

- **Click row**: expand in place (accordion, grid-rows transition) → reveal lineups, events timeline, stats tabs. Second click collapses.
- **Hover row**: subtle bg shift + chevron brightens. Keyboard: arrow keys navigate rows, Enter expands, Esc collapses.
- **Sidebar league click**: filters feed, updates URL (`/leagues/[slug]`), preserves WS subscription diff.
- **Theme toggle**: respects prefers-color-scheme, persists in localStorage, no flash on load (inline script).
- **Score-change feedback**: visual flash + `aria-live="polite"` announcement ("Arsenal 2–1 Chelsea, 67 minutes"). Reduced-motion → color tick only, no bg animation.
- **Focus**: 2px purple ring, offset 2px, visible on all interactive surfaces.

## 7. Content Requirements

- **Header**: logo "PulseScore", search placeholder "Search team, match, league", theme toggle a11y label.
- **Sidebar**: sport group label ("Football"), league entries (name + country flag + live count badge).
- **League header**: crest, league name, season ("2025/26"), live count ("3 live").
- **MatchCard**: team names (truncate at 18ch, tooltip on truncation), score ("2 – 1", en-dash), minute ("67'"), status pills ("LIVE", "HT", "FT", "15:00").
- **Empty**: "No matches today" + next fixture ISO time formatted to user locale.
- **Error**: "Couldn't refresh scores — retrying in 3s." + manual retry.
- **Disconnect**: "Reconnecting to live feed…" + attempt count.

Dynamic ranges: team name 3–30 chars, score 0–9 per side typical (99+ edge, never overflow), minute 1'–120'+ (extra time "90+3'").

## 9. Resolved Decisions

1. **Featured hero band**: YES. Top of feed shows 1 featured live match (biggest live, else next kickoff) in larger variant with subtle pitch-grass background image. Uniform rows below.
2. **Timezone**: server-default (Europe, from `.env` `TZ`). No user picker. Format via `Intl.DateTimeFormat` with server TZ.
3. **Favorites**: IN SCOPE. Star icon per row → localStorage (MVP, no auth dependency). Sidebar gets "⭐ Favorites" pinned group above leagues. Favorited matches also pinned to top of their league section.
4. **Goal toast**: IN SCOPE. Unfocused-tab detection → title flash ("⚽ Arsenal 2–1 Chelsea") + optional sound (opt-in, muted default, stored pref). In-focus toast slide-in from top-right, auto-dismiss 4s, click → jump to match.
5. **Odds**: IN SCOPE. MatchCard row reserves right-side column for 1X2 odds (3 compact pills). Hidden on narrow container via `@container`.
6. **Drill-down**: BOTH. Click row → expand in place. Click team/score area → `/match/[id]` full page. Chevron = expand-only affordance.
7. **Freshness**: per-row. Tiny muted "·2s" next to status pill, updates from WS `lastTick`. Header keeps global connection dot only.

## 10. Updated MatchCard Row Budget

`[favorite ★] [time/minute+freshness] [home team+logo] [SCORE] [away team+logo] [status pill] [1X2 odds] [chevron]`

Container-query breakpoints:
- `≥720px`: full row as above.
- `520–719px`: hide odds pills (tap row to reveal in expand).
- `<520px`: stack team names above score, time left-rail.
